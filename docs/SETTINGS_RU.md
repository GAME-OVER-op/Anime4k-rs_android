# Описание настроек

## Режим

- `legacy` — старый стабильный Anime4K-rs алгоритм.
- `a` — чистое 1080p аниме, хороший первый выбор.
- `b` — мягче, лучше для 720p или алиасинга.
- `c` — шумное/низкое качество, включает denoise-подход.
- `aa`, `bb`, `ca` — более сильные цепочки, медленнее.

## Масштаб

- x2 — 1080p → 4K.
- x4 — 480p → примерно 2K/4K, но медленно.

## Quality

- `fast` — быстрее.
- `balanced` — базовый баланс.
- `high` — лучше качество.
- `ultra` — сильнее обработка, медленнее.

## Push Gradient

Главный параметр для контуров. Если переборщить — могут появляться жёсткие края.

## Push Color

Подтягивает цветовые области. Слишком большое значение может замылить детали.

## Denoise

Убирает шум и сжатие. Полезно для старого/низкобитрейтного видео.

## Deblur

Повышает резкость. На сильных значениях может создавать ореолы.

## Затемнение линий

Делает контуры глубже. Часто даёт эффект «4K remaster».

## Утончение линий

Осветляет края толстых линий. Лучше использовать осторожно.

## Защита светлых областей

Уменьшает выбросы яркости и ringing после усиления резкости.


## Расширенные числовые настройки

В интерфейсе у числовых параметров теперь есть два способа ввода:

- ползунок для безопасного диапазона;
- поле ручного ввода справа.

Если нужно значение выше диапазона ползунка, его можно ввести вручную. Приложение покажет предупреждение, что это экспертное значение вне диапазона ползунка.

## Производительность видео

RAM-буфер кадров не используется — оставляем файловый pipeline как в старом `anime4k_video.sh`.

Добавлены настройки под будущий видео-экран:

- одновременных upscale-процессов;
- выходной FPS;
- режим FPS: `keep`, `simple`, `interpolate`;
- CRF;
- preset кодека;
- удаление исходных PNG после upscale;
- пауза каждые N кадров;
- длительность паузы.

Параллельность не меняет качество картинки, а только запускает несколько независимых обработок кадров одновременно.
