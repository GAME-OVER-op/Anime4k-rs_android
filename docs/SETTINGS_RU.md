# Настройки приложения

В этой версии оставлены только реальные настройки, которые используются классическим Anime4K-rs и видео-пайплайном FFmpeg.

## Масштаб

Аналог CLI-параметра:

```text
anime4k -s
```

Обычно:

- `2` — 1080p → 4K;
- `4` — для слабого исходника, но медленнее.

## Количество прогонов

Аналог:

```text
anime4k -i
```

Старый хороший дефолт: `3`.

Чем больше прогонов, тем сильнее обработка, но тем медленнее.

## Push Gradient

Аналог:

```text
--pgs
```

Главная сила восстановления контуров. Обычно `1`.

## Push Color

Аналог:

```text
--pcs
```

Подтягивание цвета. Обычно `0`.

## Одновременных upscale-процессов

Сколько кадров обрабатывать одновременно.

- `1` — как старый скрипт, стабильно.
- `2` — хороший старт.
- `3–4` — для мощного телефона.
- больше — вручную, если хватает охлаждения, памяти и скорости накопителя.

Параллельность не меняет качество, только скорость.

## Выходной FPS

- `0` — оставить FPS оригинала.
- любое число — использовать как целевой FPS для режимов `simple` или `interpolate`.

## Режим FPS

- `keep` — не менять FPS.
- `simple` — быстрый `ffmpeg -r`, кадры могут дублироваться.
- `interpolate` — `minterpolate`, плавнее, но очень тяжело для телефона.

## CRF

Качество кодирования libx264.

- меньше = лучше качество и больше файл;
- обычно `16–20`;
- дефолт `18`.

## Preset

Скорость кодирования ffmpeg:

```text
ultrafast, superfast, veryfast, faster, fast, medium, slow
```

## Удалять исходные PNG

Экономит место после успешного upscale кадра.

## Оставить рабочую папку

Полезно для отладки и продолжения, но занимает много места.

## Паузы для охлаждения

- `Пауза каждые N кадров`: `0` = без пауз.
- `Длительность паузы`: сколько секунд ждать.
